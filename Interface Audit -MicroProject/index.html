In my micro project, the “blind box” I created reflects several key principles discussed by Norman and Nielsen, specifically in areas such as feedback, signifiers, and system visibility, while also demonstrating areas where usability can be improved. At first glance, the design appears to be minimal with a simple layout. The main interaction is a cube that is centered labeled “PRESS ME” which acts as the primary signifier for the interaction. This aligns with Norman’s idea that users should be able to understand possible actions through clear cues. However, beyond this, the user lacks strong visibility of system status. The interface does not communicate what stage of the game the user is at. Due to this, users may feel uncertain about what phase of the game they are in or what effect their actions have. This relates to Nielsen’s heuristic which emphasizes the importance of keeping users informed through clear and immediate feedback. Another key miscommunication is the relationship between elements on the page. The shapes at the top left of the screen indicate targets, but this connection is not explicitly stated in the structure of the current layout. This creates weak mapping between the user’s actions (pressing the cube) and the intended outcome (matching the shapes), leaving the users to form their own assumptions about how the system works. In regards to Norman’s framework, this demonstrates a gap in the conceptual model, since the system does not clearly guide how different components relate to one another. Instead, users must rely on trial and error, which can also reduce the clarity of the interaction. Even though there is some text on the screen informing the basics of the game, the design also assumes that users will intuitively understand that only the cube is interactive, while the other shapes are static elements. This may cause uncertainty in affordances and signifiers, as the shapes visually correspond to interactive elements but do not respond to user input. According to Nielsen’s heuristics, this can be seen as a violation of both visibility and consistency, as users may question if they are missing a possible action. While instructional text can provide some guidance, it requires users to read and interpret information rather than allowing the interface itself to communicate behavior through structure and feedback. For design improvements, several structural changes could be made. For example, establishing a clearer hierarchy by creating distinct labels separating the interface such as “Target Shapes,” “Interaction Area,” and “Progress” would help users better understand the system at a glance. Second, introducing a visible system status like a progress indicator would provide continuous feedback and strengthen the user’s sense of progression. Finally, clarifying the interaction by directly labeling the cube as the only clickable element and visually distinguishing it from the non-interactive elements would reduce confusion and improve usability. Overall, while the blind box design is a fun idea, engaging, and intentionally minimal, aligning it more closely with Norman’s principles and Nielsen’s heuristics would allow the experience to feel more intuitive, guided, and responsive.
